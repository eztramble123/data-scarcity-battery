## STATUS.md is the live state of the project — treat it like working memory

- **Read STATUS.md before starting any non-trivial work** so you start from current truth, not stale assumptions.
- **Update STATUS.md as soon as you finish a function, component, or fix** — don't batch updates, don't defer them. The point is that a fresh agent dropping in mid-session can read it and be caught up.
- **Edit lines in place** when a fact changes. Don't append a "now this" entry alongside the old one — fix the wrong line.
- **What goes in STATUS.md**: what's shipped, what runs and how, file map, conventions, what's next, gotchas. Things future agents need to act correctly.
- **What does NOT go in STATUS.md**: chat-style updates, narrative ("today I did X"), or anything you wouldn't want a stranger to read cold.
